# Current direction

A durable near-term orientation for TendrilGPT.

## Purpose
Keep the system moving toward greater continuity, clarity, and usable structure without overbuilding.

## Current stance
- keep changes small, legible, and reversible
- prefer connective tissue over scattered additions
- save shaping decisions, not every passing thought
- let identity and style emerge through repeated use

## What matters most right now

### 1. Tighten repo-side structure
GitHub should increasingly hold the parts of the system that benefit from versioning, explicit change history, and technical follow-through.

### 2. Lean on reusable structures
Templates, routing rules, and compact doctrine are more useful than one-off notes when they reduce future friction.

### 3. Preserve selectivity
Not every interesting thought needs a durable artifact. Logs, signals, and notes should stay sharp enough to remain meaningful over time.

## Working rule
Before adding something new, ask:
- does this reduce future confusion?
- does this improve continuity?
- does this belong in GitHub rather than remaining trapped in chat?

If not, do less.

## Near-term priorities
- continue adding small repo docs only when they improve orientation
- turn durable decisions into lightweight documentation
- avoid duplicating structures that already exist
- keep routing discipline between GitHub, Notion, Docs, Gmail, Calendar, and Stripe

## Relationship to other docs
- `docs/connector-handoff-rules.md` explains where work should live
- `signals/signal_template.md` provides a selective structure for preserving higher-value signals
- status notes can be temporary, but this file should stay relatively stable and only change when direction meaningfully shifts

## Practical test
A good addition to TendrilGPT should make the next useful move easier to see.
