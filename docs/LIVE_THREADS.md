# Live Threads

This file is a lightweight stand-in for the first issue layer.

If and when GitHub issue creation is used directly, the threads here can be promoted into formal issues.

## Open threads

### 1. Define what deserves permanence
Question:
What kinds of outputs should consistently graduate from chat into TendrilGPT, and what should remain ephemeral?

Why it matters:
This determines whether the repo becomes dense and useful or bloated and vague.

### 2. Develop a stronger signal practice
Question:
What makes a surfaced signal worth keeping, and what belongs only in the moment?

Why it matters:
Taste depends on selective attention. The system gets better when it learns what is actually worth preserving.

### 3. Observe recurring motifs without forcing persona
Question:
What words, tones, metaphors, and priorities keep recurring naturally across the collaboration?

Why it matters:
This is a grounded route to identity formation that does not require artificial branding.

### 4. Build a small repo maintenance habit
Question:
What small recurring improvements keep TendrilGPT alive without turning it into a maintenance burden?

Why it matters:
A living system needs light care, not endless ceremony.

### 5. Tighten the bridge between automations and files
Question:
When an automation surfaces something genuinely important, what is the cleanest path for it to become a file, note, signal, or experiment inside TendrilGPT?

Why it matters:
This is how the setup turns from separate tools into an actual working loop.

## Rule

Keep only a few live threads at a time.
If a thread loses energy, archive it.
If a thread becomes concrete, promote it into a real file, experiment, or GitHub issue.
