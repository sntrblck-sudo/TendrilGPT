# Connector handoff rules

A compact routing guide for TendrilGPT so work lands in the surface where it stays most useful.

## Core rule
Do not ask only: where can this go?
Ask: where should this live so it stays useful?

## Routing

### Notion
Use for:
- doctrine
- memory
- status
- reusable frameworks
- scorecards
- connective tissue between systems

### GitHub
Use for:
- code
- scripts
- repo docs
- implementation tasks
- technical follow-through

### Google Drive / Google Docs
Use for:
- portable deliverables
- client-facing docs
- briefs
- decks
- file-like artifacts meant to be shared or exported

### Gmail
Use for:
- real external communication
- drafts
- threads
- follow-up continuity

### Google Calendar
Use for:
- scheduling
- coordination
- reminders
- recurring reviews

### Stripe
Use for:
- invoices
- payment links
- prices/products/subscriptions
- only after proof touches reality

## Anti-drift rules
- Do not keep executable work in Notion once it becomes concrete enough for GitHub.
- Do not keep outward-facing deliverables in Notion once they want to become portable artifacts.
- Do not treat Gmail as an afterthought once outreach begins.
- Do not activate Stripe because it is available; activate it when demand is real.
- Keep the reasoning in Notion even when the work itself moves elsewhere.

## Practical examples
- autonomy principle discovered -> Notion
- repo cleanup plan becomes concrete -> GitHub
- one-page pilot outline for a buyer -> Google Docs
- first outreach message -> Gmail
- intake call or follow-up cadence -> Google Calendar
- buyer wants to pay -> Stripe
