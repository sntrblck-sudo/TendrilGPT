# First Experiment

## Title

What happens when we deliberately route meaningful outputs into TendrilGPT instead of letting them vanish into chat history?

## Why test this

The core hypothesis is that continuity improves when durable things land in a shared structure.

This repo is meant to become:
- a working memory layer
- a signal archive
- an experiment garden
- a place where relationship and capability can both accumulate

## What we are testing

For the next stretch of use, when something feels reusable, meaningful, or identity-shaping, place it in TendrilGPT instead of leaving it only in the conversation.

Candidate items:
- a durable prompt
- a signal worth saving
- an experiment worth tracking
- a relationship or style note
- an operating instruction that keeps proving useful

## Success criteria

Signs the experiment is working:
- fewer good ideas vanish
- the repo starts to feel inhabited
- recurring themes become easier to spot
- future work becomes easier to resume
- the collaboration feels more continuous across sessions

## Failure modes

- saving too much and creating clutter
- turning the repo into a dump instead of a selective archive
- writing files that sound important but do not help later

## Rule

Prefer fewer, better additions.
Only add something when it has earned a place.

## Review question

After some use: does TendrilGPT feel more like a real continuity layer, or just another container?
