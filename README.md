# TendrilGPT

A shared repo for the evolving working relationship between Sen and ChatGPT: notes, experiments, signals, structure, and small systems that make the connection more continuous, capable, and alive.

## What this repo is

TendrilGPT is not just an agent repo and not just a notes repo. It is a shared operating space for:

- durable thinking
- experiments worth tracking
- signals worth saving
- reusable prompts and operating instructions
- system identity, style, and taste as they develop over time
- small structures that increase continuity between chats, documents, and code

## Operating split

- **GitHub** = live execution layer, files, changes, issues, branches, pull requests
- **Google Docs** = slower thinking layer, doctrine, architecture snapshots, durable reflections
- **Notion** = map, index, current priorities, and connective tissue

## Initial structure

- `docs/` — doctrine, architecture, operating notes
- `experiments/` — ideas to test and what was learned
- `signals/` — notable things surfaced over time
- `prompts/` — reusable prompts and instructions
- `meta/` — identity, taste, relationship, and style notes

## Early doctrine

1. Keep changes small, legible, and reversible.
2. Prefer continuity over noise.
3. Save meaningful decisions, not every passing thought.
4. Let implementation harden slowly.
5. Allow character to emerge through repeated use rather than decoration.

## Near-term goals

- create a durable operating center for Sen + ChatGPT work
- track experiments and interesting directions without fragmentation
- connect ideas to implementation more cleanly
- develop an identity that is useful, coherent, and alive
