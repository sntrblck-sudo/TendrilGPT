# Signals Workspace Guide

Status: active
Date: 2026-03-30

## Purpose
This folder is the durable landing zone for signals worth preserving.
A signal is not just anything noticed. It is an observation, pattern, tension, opportunity, or direction that appears meaningful enough to matter later.

## What belongs here
Use this folder for signals that are:
- high-signal
- actionable
- structurally useful
- likely to compound
- repeated enough to matter

Examples:
- recurring bottlenecks
- promising operator patterns
- repeated user preferences that affect system behavior
- useful shifts in the operating picture
- notable lessons from repeated work

## What does not belong here
Do not use this folder for:
- every passing thought
- low-value chatter
- decorative ideas with no operational consequence
- duplicates that add no clarity
- raw exhaust from every loop cycle

## Suggested use
When a signal deserves durable capture:
1. start from `templates/signal-entry.md`
2. create a compact signal note in this folder
3. promote it into docs or implementation work only if it crosses the promotion threshold

## Naming guidance
Prefer short, legible names.
Examples:
- `resume-from-durable-state.md`
- `small-safe-moves-compound.md`
- `operator-kernel-needs-front-door.md`

## Folder rule
This folder should stay selective.
If signals are accumulating without shaping action, the standard for entry is too low.
